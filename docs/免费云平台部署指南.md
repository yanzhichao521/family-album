## 方案一：Render（最推荐）

### 优点
- ✅ 免费额度：750小时/月（足够24小时运行）
- ✅ 支持前后端分离部署
- ✅ 自动 HTTPS
- ✅ 自动部署
- ✅ 稳定可靠

### 部署步骤

#### 1. 准备工作

```bash
# 确保代码已提交到 GitHub
git init
git add .
git commit -m "Initial commit"
# 推送到 GitHub 仓库
```

#### 2. 注册 Render

访问 https://render.com/ 注册账号（推荐用 GitHub 账号登录）

#### 3. 部署后端

1. 点击 "New +"
2. 选择 "Web Service"
3. 连接你的 GitHub 仓库
4. 配置：
   - **Name**: `familyalbum-backend`
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main.py`
   - **Instance Type**: `Free`
5. 环境变量：
   - `PYTHON_VERSION`: `3.10`
6. 点击 "Create Web Service"

等待部署完成（约5-10分钟），获得后端地址，如：
`https://familyalbum-backend.onrender.com`

#### 4. 部署前端

1. 再次点击 "New +"
2. 选择 "Web Service"
3. 连接同一个 GitHub 仓库
4. 配置：
   - **Name**: `familyalbum-frontend`
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Start Command**: `npm run preview`
   - **Instance Type**: `Free`
5. 环境变量：
   - `VITE_API_URL`: `https://familyalbum-backend.onrender.com`
6. 点击 "Create Web Service"

等待部署完成，获得前端地址，如：
`https://familyalbum-frontend.onrender.com`

#### 5. 完成！

访问前端地址，分享给朋友即可！

## 方案二：Railway（配置最简单）

### 优点
- ✅ 免费额度：$5/月（够用）
- ✅ 一键部署
- ✅ 支持前后端
- ✅ 自动 HTTPS

### 部署步骤

#### 1. 注册 Railway

访问 https://railway.app/ 注册账号

#### 2. 创建项目

1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择你的仓库

#### 3. 部署后端

1. Railway 会自动检测到 Python 项目
2. 配置：
   - Root Directory: `backend`
   - Command: `python main.py`
3. 点击 "Deploy"

#### 4. 部署前端

1. 在项目中点击 "New Service"
2. 选择 "Deploy from GitHub repo"
3. 选择同一个仓库
4. 配置：
   - Root Directory: `frontend`
   - Command: `npm run build && npm run preview`
5. 环境变量：
   - `VITE_API_URL`: 后端地址
6. 点击 "Deploy"

#### 5. 完成！

## 方案三：Vercel（前端最佳）

### 优点
- ✅ 前端完全免费，无限流量
- ✅ 全球 CDN，速度快
- ✅ 自动 HTTPS
- ✅ 部署极快

### 部署步骤

#### 1. 部署前端到 Vercel

1. 访问 https://vercel.com/ 注册
2. 点击 "New Project"
3. 导入 GitHub 仓库
4. 配置：
   - Root Directory: `frontend`
   - Framework Preset: `Vite`
5. 环境变量：
   - `VITE_API_URL`: 后端地址（见下方）
6. 点击 "Deploy"

#### 2. 部署后端到 Render 或 Railway

使用方案一或方案二部署后端

#### 3. 完成！

## 注意事项

### 1. 免费额度限制

- **Render**: 750小时/月 ≈ 24小时/天 × 31天
- **Railway**: $5/月，通常够用
- **Vercel**: 前端完全免费
- **Zeabur**: 500小时/月

### 2. 睡眠问题

部分免费平台会在无流量时让服务休眠，首次访问需要等待几秒唤醒。这是正常的。

### 3. 数据持久化

免费平台重启后数据会丢失，建议：
- 使用外部数据库（如 Supabase，有免费额度）
- 定期备份数据库

### 4. 文件存储

上传的文件需要使用对象存储：
- **Cloudflare R2**: 免费额度大（10GB）
- **Backblaze B2**: 10GB 免费存储
- **Supabase Storage**: 1GB 免费
